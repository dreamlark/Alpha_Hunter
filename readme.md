

# ALPHA HUNTER – 微信公众号财经资讯模板

## 🎯 项目简介
`ALPHA HUNTER` 是一套面向微信公众号的金融资讯日推模板。以经典“深海鎏金”视觉效果为基础，是从“信息展示”到“可执行、可复盘、可纠错”的完整内容框架和决策支撑系统。

├──cover.html   			# 封面模板（占位符版）

└── README.md                    # 本文件

**注意事项**  

- 模板中的 `{{变量名}}` 占位符需在实际发送前替换为真实内容。
- 模板未引用任何外部 CSS 或图片，所有样式内联，完全独立。
- 已在 `<body>` 中声明 `color-scheme:light only;`，并配合白色边框等手段防止微信暗黑模式颜色反转。

## 🚀 快速开始
### 1. 填充变量
对照下方 **变量字典**，将模板内所有 `{{VARIABLE}}` 替换为实际内容。颜色变量（如 `{{DJ_COLOR}}`）直接填入 CSS 颜色值（例如 `#cc3333` 代表红涨，`#228b22` 代表绿跌）。

### 2. 生成预测表格行
`{{PREDICTION_ROWS}}` 是循环块，需要按以下格式生成每一行：
```html
<tr>
  <td style="background-color:#f8f9fc; padding:6px 8px; border:none; border-bottom:1px solid #e8e8e8;">
    <span style="color:#444444; font-size:12px;">早盘</span>
  </td>
  <td style="background-color:#f8f9fc; padding:6px 8px; border:none; border-bottom:1px solid #e8e8e8; word-break:break-all;">
    <span style="color:#333333; font-size:12px;">高开+30分钟量>800亿 → 维持3成仓；低开破3180 → 减至1成</span>
  </td>
  <td style="background-color:#f8f9fc; padding:6px 8px; text-align:center; border:none; border-bottom:1px solid #e8e8e8;">
    <span style="color:#228b22; font-size:12px;">75%</span>
  </td>
</tr>
```

每一行交替使用 `#f8f9fc` 和 `#ffffff` 作为背景色，最后一行的第一个和最后一个 `td` 需设置 `border-radius:0 0 0 4px` 与 `border-radius:0 0 4px 0`。

### 3. 发送
在微信公众号后台“新建图文”中，点击“源代码”按钮，粘贴完整 HTML。务必在正式发送前**在不同机型、不同模式（深色/护眼）下预览**。

## 🎨 设计体系
### 色彩语义
| 用途           | 颜色                              | 说明                                         |
| -------------- | --------------------------------- | -------------------------------------------- |
| 头图背景       | `#0d1f3c`                         | 深海蓝，配 `border:1px solid #ffffff` 防反转 |
| 标题/品牌色    | `#ffd666`（金）                   | 核心标识                                     |
| 上涨/积极      | `#cc3333` 或 `#228b22` 由后端决定 | 遵循国内涨红跌绿习惯                         |
| 卡片底色       | `#eef2ff` / `#f8f9fc` 等          | 浅色系，保持清爽                             |
| 四象限卡片边框 | `rgba(0,0,0,0.08)`                | 在护眼模式/高对比度下保留卡片边界            |
| 总结/锦囊卡片  | `#fffdf0` 背景 + `#ffd666` 左边框 | 统一视觉锚点                                 |
| 导航彩条       | `#e6b800` 左侧边框                | 区别正文区域                                 |

### 间距与留白
- **模块间** 8px：通过 `margin-top:8px` 实现，不使用空 `<td>`，避免被微信折叠。
- **卡片内** 10-14px 内边距，保证呼吸感。
- 文字行高 1.5-1.85，提升可读性。

### 字体层级
- 模块标题：18px 深色
- 卡片主标题：15-16px
- 正文/数据：13-14px
- 辅助说明：12px
- 免责声明：11px  
  所有字号 ≥12px，规避微信强制放大导致的排版错乱。

### 卡片规范
- 圆角半径：6px（次级卡片）或 16px（头图）
- 左侧强调条：`border-left:3px solid` 用于总结、锦囊、导航
- 边框：`border:1px solid rgba(0,0,0,0.08)` 适用于所有四象限卡片和外盘指标卡片，防止背景色在系统主题下被剥离后看不见。

## 🖼️ 封面图生成

### 文件说明
- **`cover.html`**：极简品牌封面 HTML 模板（变量占位符版）。使用 Chrome 浏览器打开，通过“捕获节点截图”即可生成 900×383 像素、背景透明的 PNG 封面图，直接用于微信公众号封面。

### 设计思路
- **品牌优先**：巨大的 `Alpha Hunter` 金色字母与 `🎯` 图标占据视觉中心，在订阅号列表中形成强辨识度。
- **一句话钩子**：仅保留一句核心判断，左侧金色竖线引导阅读，信息量极低但内容吸引力极高。
- **极简背景**：深海蓝底色配合微弱金色光晕，无多余装饰，突出文字内容。
- **透明输出**：封面元素外层无背景色，截图后得到纯 900×383 封面图，无黑边或白边干扰。

### 变量字典
| 变量                   | 说明                       | 示例                      |
| ---------------------- | -------------------------- | ------------------------- |
| `{{COVER_HOOK_LINE1}}` | 钩子文案第一行             | `外围回暖，A股缩量收官。` |
| `{{COVER_KEYWORD1}}`   | 关键词一（金色高亮）       | `AI算力`                  |
| `{{COVER_CONNECT}}`    | 关键词连接词               | `与`                      |
| `{{COVER_KEYWORD2}}`   | 关键词二（金色高亮）       | `低空经济`                |
| `{{COVER_HOOK_LINE2}}` | 钩子文案第二行（关键词后） | `双主线不变。`            |
| `{{COVER_DATE}}`       | 日期                       | `2026.04.30`              |
| `{{COVER_ISSUE}}`      | 期号                       | `088`                     |

### 使用步骤
1. 根据当天核心内容，填写上述变量。
2. 用 Chrome 打开 `cover.html`。
3. 按 `F12` 打开开发者工具 → 按 `Ctrl+Shift+C` 选择 `<div class="cover">` 元素。
4. 在“元素”面板右键该节点 → **“Capture node screenshot”**。
5. 浏览器自动下载背景透明的 900×383 PNG 文件，上传至微信公众号封面即可。

### 每日维护
仅需修改 `.hook` 文案和 `.date` 行，无需调整样式。

## 🧪 测试与兼容性
### 发送前必测
- iOS 深色模式：头图不应反转为灰蓝，金色文字不应变土黄。
- 安卓护眼模式：四象限卡片是否仍有清晰边框可见。
- 不同屏幕宽度 (iPhone SE ~ 15 Pro Max)：模块是否水平居中且不溢出。
- 折叠屏展开态：模块宽度不应过窄（已用 `width:94%` 适配）。
- 外盘六卡片：涨跌颜色是否与预期一致。

### 已知浏览器差异
- 微信/企业微信内置浏览器对 `border-collapse` 处理稳定，但 `<td>` 内 `border-radius` 在极个别安卓机型有失效风险；已通过嵌套 table 规避。

## 🛠️ 数据管道建议
1. 后端脚本从行情 API、数据库获取数据后，按变量字典生成键值对。
2. 颜色值由涨跌方向判断：涨 `#cc3333`，跌 `#228b22`，平 `#000000`。
3. `{{PREDICTION_ROWS}}` 用模板引擎循环输出（如 Jinja2、Mustache）。
4. 所有文本内容应由分析师撰写或 AI 辅助生成后人工审核，特别是反方观点和预期差，不可使用泛泛的风险提示。

## 📋 常见问题
**Q: 为什么导航没点击跳转？**  
A: 微信公众号图文不支持页内锚点，导航仅作为阅读指引，无法实现跳转。

**Q: 能否添加更多主线？**  
A: 限于移动端阅读体验，主线数量控制在 2 条。如需展示第三条，可精简到 2 条后在风险总结中提及。

**Q: `{{CONSENSUS_DIVERGENCE_RULE}}` 怎么写？**  
A: 如实说明本模板的三维分歧处理逻辑。如：“三维共振全指向谨慎则轻仓；若出现 2:1 分歧，优先遵循游资信号，因游资对短线风险更敏感。”

**Q: 执行评分标准如何定义？**  
A: 建议基于以下规则自动生成：
- 触发信号后 5 分钟内完成操作：+1
- 实际仓位与计划偏差 ≤5%：+1
- 未按计划止盈/止损：-1  
满分 2 分，≥1 分为合格。
